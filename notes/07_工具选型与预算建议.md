# 07_工具选型与预算建议

> 适合人群：已经理解工作区思路，准备开始选工具的同学  
> 使用建议：这篇偏“选型参考”，不用一上来就全部读完；如果你还没跑通第一轮任务，建议先回去看 `01_零基础三步走.md`

## 先说结论
如果你现在还是第一次系统接触 AI，我最建议的顺序是：

1. 先用 A 类把完整流程跑通。  
2. 真遇到上限，再考虑 B 类。  
3. C/D 类更多是“有特定需求时再试”，不是第一站。

## GitHub Copilot 学生福利

如果你有学校邮箱，这里还有一个很适合学生顺手开通的福利：**GitHub Copilot**。

它和 Claude Code 不完全一样。Claude Code 更像“能和你对话、帮你处理工作区任务的 AI 搭子”，而 GitHub Copilot 更像“你打字时在旁边帮你续写和补全的助手”。两者可以配合用，但不一定要一开始就全都装上。

如果你想低成本试试，可以这样走：

1. 准备好学校邮箱（常见如 `xxx@edu.cn`）。
2. 打开 [education.github.com/pack](https://education.github.com/pack) 申请学生权益。
3. 验证通过后，在 VSCode 扩展市场里安装 `GitHub Copilot` 并登录。

它比较适合这些场景：

1. 记笔记时自动补全文字。
2. 写提纲时帮你续写下一段。
3. 写简单代码、注释、模板时减少重复输入。

> ⚠️ GitHub 网站在国内有时访问不稳定。如果打不开，可以先跳过，不影响主流程。等网络条件更合适时再回来开通就行。

## 最后提醒
选工具这件事，很容易让人越看越乱。  
如果你现在还没有真正跑通过一次“作业要求 -> 材料 -> 初稿 -> 改稿 -> 终稿”的闭环，那先别在工具比较上花太久时间。

先把事做成，再慢慢升级工具。
