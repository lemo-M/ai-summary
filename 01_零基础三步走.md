# 01_零基础三步走：让 AI 真正帮你做完学习任务

> 作者：mong  
> 最后更新：2026-03-17  
> 适合人群：大学生、文科生、第一次系统接触 AI 的同学

## 先说人话结论
很多同学不是不会用 AI，而是一直在“聊天模式”里打转。  
你真正需要的是：把 AI 放到你的学习资料旁边，让它和你一起做任务。

这篇文档的目标只有一个：  
把你从“问一句答一句”，带到“能稳定做出作业/论文成果”。

### 先看一个真实变化
我前段时间带过一个文科生朋友入门 AI。  
她一开始最常见的用法，就是在豆包、Gemini 这类聊天框里反复开新对话：

1. 这一轮让 AI 帮她写大纲。  
2. 下一轮又要重新解释老师要求。  
3. 再下一轮，AI 已经忘了前面改到哪里。  

后来我让她换了一个思路：

1. 用 VSCode 打开一个专门的作业文件夹。  
2. 把“作业要求、课堂笔记、参考文献”都放进去。  
3. 再让 Claude Code/Codex 这类终端 AI 直接围着这个工作区做事。  

她最明显的感受不是“AI 变聪明了”，而是：  
**AI 终于开始围着资料持续工作，而不是每次都像第一次见面。**

---

## 一、我们最常见的学习痛点

1. 写论文时，资料很多，最后结构还是乱。  
2. 每次开新对话都要重新解释背景，特别费时间。  
3. 老师临时改要求，前面内容基本白做。  
4. AI 写得看起来不错，但一细看就空。  
5. 截止日期快到了，任务多，脑子会炸。

如果你也中 2 条以上，说明你缺的不是“更会提问”，而是“更稳的做事流程”。

> 💡 **核心概念1：AI 幻觉（它会“一本正经地胡说八道”）**  
> 纯聊天模式下，AI 很容易乱编数据或文献。但在工作区里，因为你把真实材料喂给了它，它就能“照本宣科”，你也能直接对照原文快速核验事实。

## 二、为什么我推荐 VSCode + Claude Code + CCSwitch

我现在是在武汉互联网公司实习,也使用过很多ai编程的工具。  
这套组合对新手真的友好，站在一个学生的角度去思考过很多方式，觉得这套流程是既有思考过程又能真实体验一些安装过程的（在安装过程中，学会ai辅助），我也已经带过身边好几个朋友跑这套流程，大家普遍反馈都是：

1. 上手没有想象中难。  
2. 一旦跑通，会明显感觉 AI 不再只是聊天，而是真的开始帮你做事。  
3. 用这套流程去做论文、改稿、整理资料，会更容易建立起“AI 到底怎么参与真实任务”的基础认识。  

除了学习任务，我也用过“改简历”的 skill 去帮考研复试、秋招、找实习的朋友写简历和改简历，包括后续的准备面试都可以使用对应的skill去完成。  
很多人原来不是简历内容完全不行，而是表达太散、重点不突出、岗位方向不够清楚。  
用工作区 + skill 的方式去改，效果会比单纯在聊天框里问“帮我润色一下简历”稳定很多。

### 1）先说 VSCode
你就把它当成“好用的文本编辑器”就行，不用一上来把它想得很复杂。  
它对学生最实用的点是：

1. **插件生态丰富**：它不仅有支持 Claude Code 的诸多可视化插件（让你不用看着黑乎乎的终端敲代码），还能直接使用 GitHub Copilot 自带的强大 AI（可以用大学生的教育邮箱免费申请额度白嫖体验）。
2. **读写一体**：读文档、整理内容、改稿都在同一个窗口完成。
3. **工作区概念清晰**：资料不容易乱，AI 也能更准地抓取内容。

### 2）再说 Claude Code
它能接的 API 选择比较多，你可以按预算选方案。  
而且很多同学会走国内 API，网络门槛更低，对小白更友好。

### 3）CCSwitch 的作用
它的核心价值很简单：帮你把终端配置管理得更顺。  
比如切换配置、管理连接、减少“环境配好了又乱掉”的情况。

## 三、先看一眼工作区树（先有全局感）

很多同学平时也会整理电脑文件夹，这个习惯很好。  
但“AI 学习工作区”的整理方式有一点不同：

1. 平时整理：更像“按时间、按课程名、按习惯”收纳。  
2. 工作区整理：更像“按任务流程、按功能分工”组织。

类比编程里的做法就是：同一块业务的代码放在一起。  
在学习里也是一样：同一项作业相关的材料、过程、结果放在一起。

你先记住这个树（文件夹层级）：

```text
study-ai-workspace/
  input/               # 原始材料区（老师给的要求、你的笔记、文献）
    assignment.md
    notes.md
    references.md
  output/              # 结果区（开题报告、中期报告、终稿）
    开题报告.md
    中期报告.md
    终稿.md
  task.md              # 本次任务说明（你让AI做什么）
```

这个“树”的价值：

1. AI 每次都能快速找到材料，不用你反复解释。  
这也是 RAG 的基础思路：先从你给的资料里找，再生成内容。  
2. 你知道哪是输入、哪是输出，不会文件乱飞。  
3. 下次做同类任务，可以直接复制这套结构复用。  
这就是 Skill 的雏形：把“好用流程”沉淀成可重复使用的模板。

> 💡 **核心概念2：上下文（它记性有限）**  
> 就像人的短期记忆有限一样，丢给 AI 的资料太多它就会“忘”。建好文件夹，每次任务只喂给它相关的核心材料，它才能跑得最准。

## 四、核心思路：给 AI 一个“学习工作区”

你可以把“工作区”理解成一个课程资料文件夹。  
这个文件夹里放：

1. 作业要求。  
2. 课堂笔记。  
3. 参考文献。  
4. AI 产出的初稿和改稿。

这样做的好处很直接：

1. AI 每次都能看到同一批材料。  
2. 你的版本变化有记录，不会越改越乱。  
 后面你可以用 Git 做版本管理，再用 GitHub 做云端备份和分享。  
3. 可以复用，下次同类任务直接套流程。

## 五、零基础三步法（今天就能用）

### 第一步：说清任务目标
不要只说“帮我写一下”，要说清楚交付物。

示例：  
请基于我的材料，先生成一份 3000 字论文大纲，包含摘要、研究问题、方法、结论，语气偏学术。

一句话：目标越清楚，结果越稳。

> 💡 **核心概念3：指令模板（Prompt）**  
> 指令越具体，结果越稳。不要只说“帮我写”，要清晰说明“基于 xxx 材料，完成 yyy 目标，输出 zzz 格式”（这就是为什么我们要写 `task.md`）。

### 第二步：把材料交给 AI
把课程要求、笔记、参考文献都放进同一个文件夹。  
然后告诉 AI：请优先基于这些文件来写。

一句话：给材料，AI 才不会乱猜。

### 第三步：按版本迭代
第一版不满意很正常，重点是二改三改。

建议做法：
1. 先出 `v1`。  
2. 你指出问题（逻辑、语气、证据不够）。  
3. 再出 `v2`。  
4. 最后你人工检查引用和事实。

一句话：好结果不是“灵光一现”，而是“多轮打磨”。



### 一个最小样例：论文任务到底怎么跑
如果你还是觉得抽象，可以直接看这个最小样例。

假设你现在要写一篇课程论文，你的工作区可以这样放：

```text
study-ai-workspace/
  input/
    assignment.md      # 老师要求
    notes.md           # 你的课堂笔记
    references.md      # 参考文献摘要或摘录
  output/
    大纲_v1.md
    初稿_v1.md
    终稿.md
  task.md
```

`task.md` 可以先写得很简单：

```text
请基于 input/ 里的材料，先帮我完成这次课程论文的第一轮工作：
1. 先提炼老师要求和评分重点
2. 给出一个 3000 字论文大纲
3. 大纲要包含：摘要、研究问题、正文结构、结论
4. 优先使用我的笔记和参考材料，不要脱离资料乱写
5. 结果输出到 output/大纲_v1.md
```

然后你在终端里和 AI 配合，一般就按这个顺序走：

1. 第一轮：先让它读 `assignment.md`、`notes.md`、`references.md`。  
2. 第二轮：让它产出 `output/大纲_v1.md`。  
3. 第三轮：你检查大纲哪里偏了，再让它继续改。  
4. 第四轮：大纲稳定后，再让它扩成 `初稿_v1.md`。  
5. 最后一轮：你人工核对事实、引用和语气，再整理成 `终稿.md`。  

你会发现，真正关键的不是“你会不会一句神 prompt”，而是：  
**你有没有把任务、材料、输出，放进同一个工作区里连续推进。**

## 六、这些词先别硬背，用学生场景理解就行

1. Prompt（提示词）  
解释：你给 AI 的任务说明。  
形象理解：像你给学习搭子下任务单，讲清楚要做什么。

2. Context（上下文）  
解释：这次任务相关的背景资料。  
形象理解：就是你的课程要求、课堂笔记、参考文献这些“题目背景”。

3. RAG（检索增强）  
解释：AI 先从你给的资料里找内容，再基于这些内容生成答案。  
形象理解：像“开卷写作，但只翻你给它的那几页资料”；你把论文和课堂资料放进 VSCode 文件夹，AI 先读这些再回答，会比它凭空写靠谱很多。

4. Workflow（工作流）  
解释：把一件事拆成固定步骤，按顺序推进。  
形象理解：像你每次做论文时，固定都会走那几步：先看要求 -> 整理资料 -> 出大纲 -> 写初稿 -> 改终稿。

5. Agent（代理）  
解释：能够根据目标连续往下做事的 AI。  
形象理解：AI 不只是等你发一句，它会顺着步骤往下推进，像一个会接着干活的搭子。

6. Skill（技能）  
解释：把某类常见任务整理成可复用的方法或模板。  
形象理解：像你整理出来的一套“万能作业模板”或“固定做法”，比如“文献综述初稿模板”“周报整理模板”“简历优化模板”。

7. Model（模型）  
解释：AI 背后真正负责思考和生成内容的核心能力。  
形象理解：像不同老师的风格，有的擅长逻辑，有的擅长表达，有的速度快但写得浅。

你现在不需要死记这些词，只要先会用场景。

## 八、执行顺序（按这个来就不会乱）

1. 先搭好上面的 `study-ai-workspace` 文件夹树。  
2. 把资料放进 `input/`。  
3. 在 `task.md` 写清目标和要求。  
4. 在 VSCode 终端中用 Claude Code/Codex 跑第一版。  
5. 迭代到 `终稿.md` 后再提交。
